# マイク音声分配ツール (MicSplitter) 設計書

## 1. システム概要

本アプリケーション「MicSplitter」は、Windows環境において、1つの入力オーディオデバイス（物理マイク）の音声をキャプチャし、2つの独立した出力オーディオデバイス（仮想マイクA、仮想マイクB。本設計ではVB-Audio Virtual Cableを前提とする）へリアルタイムかつ低遅延で転送する常駐型ツールです。さらに、キャプチャしたマイク音声をユーザーが選択した再生デバイス（スピーカーやヘッドフォン）へ折り返して出力する「モニター機能」を備えています。

### 1.1. 目標とするユーザー体験 (UX)

- **直感的な視覚化**:
  メインウィンドウを開くと、入力から出力（仮想マイクA/B、モニター）へ音声が流れる様子がフローチャートのようなUIで表示され、ルーティングや各出力の有効/無効状態がひと目で理解できる。

- **リアルタイムな状態確認**:
  - 入力・出力・モニターそれぞれのノードにボリューム（音量レベル）を可視化するレベルメーター（バー）を備え、音声が正常に入力・転送されているか視覚的に確認できる。
  - タスクトレイアイコンの色を現在の出力状態に合わせて切り替えることで、ウィンドウを閉じてバックグラウンド動作している間も現在の音声ミラー先ステータスが常に把握できる。

- **スムーズな導入支援**:
  - 出力デバイスが未設定、または対象デバイスが見つからない場合、UI上からVB-Audio Virtual Cableのダウンロードサイトへのリンクを提示し、さらにWindowsのサウンド設定（コントロールパネル）を直接呼び出すボタンを表示して導入や設定確認をスムーズに促す。

- **バックグラウンド常駐と自動起動（メモリ・CPU最小化）**:
  - ウィンドウを閉じる（Closeボタン押下）または最小化すると、eframeの `set_visible(false)` などの機能を用いてウィンドウ（GUI）を非表示化し、描画イベントをスキップさせることでリソースの消費を抑えます。tauri等の別フレームワークは使用しません。
  - レジストリ（Runキー）またはスタートアップフォルダを用いた自動起動に対応し、引数 `--autostart` 適用時はGUIを表示せずトレイに直接常駐します。なお、本アプリケーションの動作に管理者権限は不要（標準ユーザー権限で動作）とします。
  - 二重起動防止用として Named Mutex を用いた単一インスタンス制限を施し、二重起動検知時には既存プロセスへ「ウィンドウ前面化」を通知し、新規プロセスは即時終了する。
  - アプリ起動後は、手動での開始操作を待たず**自動で音声の転送を開始**します。

- **動的更新と自動リカバリー**:
  - USBの脱着などによってデバイスが再接続された場合や、PCがスリープから復帰した際、自動でオーディオデバイスを再検出・ストリーム再構築を行い、動作を正常継続する。
  - GUI上のドロップダウンで設定（デバイス等）を**選択した瞬間に**即座にストリームが再構築され反映されます。

- **ショートカット機能（音声の切り替え）**:
  - 2つの出力先（仮想マイクA、仮想マイクB）に対してショートカットキー（グローバルホットキー）を割り当てる。
  - **「トグル（排他）切り替え」を既定（デフォルト）の動作**とし、1ボタンで出力先AとBを排他的にスイッチする（設定により、個別ON/OFFモードにも変更可能とする）。
  - ホットキーの初期値は他のアプリケーションと極めて衝突しにくいキー組み合わせ（例: `Ctrl+Alt+Win+F9`（出力1切り替え）、`Ctrl+Alt+Win+F10`（出力2切り替え））を設定し、設定ファイル（`config.json`）およびGUI設定パネルから、後からユーザーが任意に変更できる仕様とする。
  - トグルモード時の初期状態は、前回の設定状態（AかBか）を保持します。初回起動時の既定は出力デバイス1（仮想マイクA）をONとします。
  - 自分用の「モニター出力」のON/OFF（ミュート/アンミュート）も設定画面やホットキーから切り替え可能とする。（キーバインドは`Ctrl+Alt+Win+F8`）
  - **入力自体のミュート**も設定画面やホットキーから切り替え可能とする。（キーバインドは`Ctrl+Alt+Win+F7`）
  - **出力1・2の排他切り替え（スワップ）** ホットキーを追加し、現在の状態に関わらず必ず「どちらか片方だけがON」の状態へ交互に切り替える。（キーバインドは`Ctrl+Alt+Win+F11`）

## 2. システムアーキテクチャ

システムは大きく分けて4つのモジュールで構成されます。

### 2.1. モジュール構成

- **UI / ウィンドウ管理 (ui_main & ui_tray)**:
  - メインウィンドウの描画（フローチャートUI、レベルメーター、デバイス選択、設定項目）。
  - タスクトレイアイコンの表示、状態に応じたアイコン色の変更、および右クリックコンテキストメニューの描画。
  - 閉じる/最小化イベントのフックによるウィンドウ非表示制御（eframeベース）。
- **設定・システム連携管理 (config & system)**:
  - 設定データ（デバイス名、ショートカットキー、スタートアップ有効/無効、モニター設定など）のローカル保存。
  - OSの標準的な自動起動（レジストリRunキー等）の登録・解除。
  - Named Mutexを用いた単一インスタンス制限（二重起動防止）および既存プロセスへのウィンドウ表示命令の送信。
  - グローバル・ショートカットキーの検知。
- **オーディオエンジン (audio_engine)**:
  - Windows API (WASAPI) を介したオーディオデバイス（入力、出力、モニター）の列挙。
  - 音声データのキャプチャとレンダリング（ストリームの管理）。
  - レベルメーター表示のための音量（RMS/Peak）計算。
  - USBデバイス脱着・スリープからの復帰を検知する自動リカバリー機構。
- **データパイプライン (pipeline)**:
  - スレッド間の音声データ転送（リングバッファ）。
  - 音声処理スレッドでの音量変更・ミュート・排他切り替えの適用。

### 2.2. 技術スタック (採用クレート案)

- **オーディオ処理**: `cpal` (WASAPIバックエンド)
- **スレッド間データ通信**: `ringbuf`
- **GUIフレームワーク**: `eframe` (`egui`)
  - 即時描画（Immediate Mode）型GUI。非表示時は更新をスキップさせることでリソース消費を抑える。
- **タスクトレイUI**: `tray-icon` (ウィンドウと独立してイベントループを回せるもの)
- **二重起動防止・通信**: `windows` (Named Mutex) ＋ 軽量な名前付きパイプ (Named Pipe) またはローカルソケット
- **グローバルホットキー**: `global-hotkey` または `mki`
- **設定シリアライズ**: `serde`, `serde_json`

## 3. オーディオ・データフロー設計

### 3.1. ストリーム構成とレベル解析

- **入力ストリーム**: 物理マイクから音声を取得。同時に音量レベル（RMS値など）を計算し、UIスレッドへ共有する。
- **リングバッファ**: 入力から出力へデータを渡すための中間バッファ。**推奨サイズとして20ms〜50ms相当のフレーム数を保持する**設計とします。
- **出力ストリーム (x2)**: 仮想ケーブルへ音声を書き込む。出力前にも音量レベルを計算しUIへ共有。個別のミュートやボリューム変更処理、トグル（排他）処理は、リングバッファからポップした後、出力ストリームに書き込む直前に行う。出力ストリームがデータ不足（アンダーラン）になった場合は無音（ゼロ）を書き込み、入力と出力の処理速度のズレを吸収する。
- **モニターストリーム**: ユーザーが指定したモニター用再生デバイス（ヘッドホン・スピーカー）へマイク音声を書き込む。独自の音量調整（モニター音量）とモニターON/OFF（ミュート）を適用する。

### 3.2. データフロー図

```
               [物理マイク]
                    │
                    ▼ (WASAPI Capture + 入力レベル計算)
              (入力ストリーム)
                    │
   ┌────────────────┼────────────────┐
   ▼                ▼                ▼
[ RingBuffer A ] [ RingBuffer B ] [ RingBuffer Mon ]
   │                │                │
   ▼                ▼                ▼
(トグル/ミュート) (トグル/ミュート) (モニター音量/ミュート)
(レベル計算)     (レベル計算)          │
   │                │                │
   ▼                ▼                ▼
[仮想マイクA]    [仮想マイクB]     [モニター用スピーカー]
```

## 4. UI仕様と設定項目

本アプリケーションは、eframe (egui) によるメインウィンドウ（GUI）を表示し、ウィンドウを閉じると非表示化してタスクトレイ（システムトレイ）に常駐します。

### 4.1. メインウィンドウ構成（GUI）

メインウィンドウには以下の要素を配置します。

- **フローチャート風ルーティング図**:
  - 左側に「入力デバイス」、右側に「出力デバイス1（仮想マイクA）」「出力デバイス2（仮想マイクB）」「モニター再生デバイス」をノードとして配置し、接続線で音声の流れを表現します。
  - 各接続線上、あるいは出力デバイスノードの近くに、音声の転送ON/OFF（有効/無効）を切り替えるスイッチ/トグルボタンを配置します。
- **レベルメーター**:
  - 入力デバイス、各出力デバイス、およびモニターデバイスの音量をカスタム描画のバーでリアルタイム表示します。
  - **ゾーン分け**: 音量に応じて安全域（0〜70%：緑）、警告域（70〜90%：黄色）、危険域（90〜100%：赤色）に色分けし、クリッピングの危険度を視覚化します。
  - **滑らかなアニメーション**: 音量上昇時は瞬時に反応し、下降時は滑らかに減衰（フォールオフ）させることで、視覚的に心地よい動きを実現します。
  - **ピークホールド機能**: 瞬間的な最大音量の位置にマーカーを残し、一定時間（約1.5秒）経過後にゆっくりと下降させることで、一瞬の音割れを確認しやすくします。
  - **ミュートの明示**: ミュート時にはバー全体をグレーアウトし、中央に「MUTED」と明記して状態を分かりやすく伝えます。
- **デバイス選択ドロップダウン**:
  - システム上に存在するオーディオデバイスを列挙し、入力/出力デバイス/モニターデバイスをドロップダウンメニューから選択できるようにします。**選択した瞬間に**即座にストリームが再構築されます。
  - リストで選択されたデバイスの右側には、OSで設定されている現在の音量レベル、サンプルレート、およびビット深度がリアルタイムで表示されます。
  - バックグラウンドで約2秒間隔の自動監視を行い、デバイスの接続や切断を検知します。使用中のデバイスが切断・無効化された場合は、ドロップダウンの表示名とラベルが即座に赤色に変化し、「(切断または無効)」と警告が表示されます。
  - 無効なデバイスが選択されていても、OSの既定デバイスへ勝手にフォールバックすることはなく、該当するストリームのみを安全に停止します。その後、デバイスが再接続されると、自動的に音声ストリームが再構築されて復旧します。
- **設定パネル（各グループは既定で展開状態）**:
  - **ショートカットキー**: 各出力・モニターデバイスへのトグル/ミュート用「グローバル・ショートカットキー（ホットキー）」の割り当てUI。
  - **デバイス設定**: 入力デバイス・各出力デバイス・モニターデバイスのドロップダウン選択。
    - 各ラベルの左側には、タスクトレイと同じ現在の設定色（後述）のアイコン（●）を表示し、対応関係を明示します。
  - **ボリュームとミュート設定**:
    - 切り替えモードの選択: 「トグル（排他）モード（既定）」または「個別ON/OFFモード」の切り替え。
    - 各デバイスの音量メーター、およびミュートON/OFF切り替えチェックボックス。モニター出力の音量スライダー。
    - 各ラベルの左側にも、デバイス設定と同様のカラーアイコン（●）を表示します。
  - スタートアップ自動起動の有効/無効チェックボックス（該当ブロック付近）。
  - Windowsのサウンドコントロールパネルを直接呼び出す「サウンド設定を開く」ボタン。

### 4.2. タスクトレイ仕様

タスクトレイのアイコン色を変更し、現在の配信（出力）状態を常時可視化します。
なお、**指定した色・配置のアイコン画像（.ico）を生成するスクリプトを別途作成し、生成したicoファイルをアプリに同梱**して動的に差し替える仕組みとします。

- **トレイアイコンの色定義（`config.json` で設定可能）**:
  - **両方OFF（完全ミュート）**: デフォルトはグレー `[150, 150, 150]`
  - **出力1のみON**: デフォルトは赤 `[255, 100, 100]`
  - **出力2のみON**: デフォルトは緑 `[100, 255, 100]`
  - **両方ON**: 左上の直角三角形が出力1の色、右下が長方形...ではなく斜め分割され右下が出力2の色になります。
  - **マイク入力ミュート時**: 現在の色の上に赤字で「×」印が重なって表示されます。

- **トレイメニュー（右クリックで表示されるコンテキストメニュー）**:
  - **設定を開く (Show)**: メインウィンドウを表示します。
  - **▶ Start / ■ Stop**: 転送の開始と停止（現在の状態に応じてトグル）。
  - --- (セパレータ)
  - **Input Device ＞**: サブメニューで入力デバイス一覧を表示（チェック付き）。
  - **Output Device 1 ＞**: サブメニューで出力デバイス1一覧を表示。
  - **Output Device 2 ＞**: サブメニューで出力デバイス2一覧を表示。
  - **Monitor Device ＞**: サブメニューでモニターデバイス一覧を表示。
  - --- (セパレータ)
  - **Exit**: アプリケーションの完全終了（プロセス終了）。

### 4.3. 設定ファイル (config.json) の構造

- アプリケーション終了時やUIでの設定変更時に状態が保存され、次回起動時に復元されます。
- 保存される主な項目: 選択デバイス名、有効/無効状態（ミュート状態）、モニター音量、切り替えモード、スタートアップ自動起動設定、ショートカットキー設定。
- **タスクトレイおよびUI表示用の色設定**: ハードコードを廃止し、以下のキーでRGB配列として設定・変更が可能です。
  - `icon_color_both_on`
  - `icon_color_out1_on`
  - `icon_color_out2_on`
  - `icon_color_both_off`
設定ファイルは `%APPDATA%\MicSplitter\config.json` に保存されます。初回起動時などファイルが見つからない場合は、以下のデフォルト値を用いて自動生成します。

```json
{
  "input_device_name": "マイク (Realtek Audio)",
  "output_device_1_name": "CABLE Input (VB-Audio Virtual Cable)",
  "output_device_2_name": "CABLE-B Input (VB-Audio Cable B)",
  "monitor_device_name": "スピーカー (Realtek Audio)",
  "input_enabled": true,
  "output_device_1_enabled": true,
  "output_device_2_enabled": false,
  "monitor_enabled": false,
  "monitor_volume": 0.8,
  "switching_mode": "toggle",
  "output_device_1_hotkey": "Ctrl+Alt+Win+F9",
  "output_device_2_hotkey": "Ctrl+Alt+Win+F10",
  "auto_start": true
}
```

※ `switching_mode` は `"toggle"` (排他切り替え) または `"individual"` (個別ON/OFF) を指定可能。

### 4.4. ログとエラーレポート

常駐アプリケーションとして安定稼働させるため、以下の仕組みを備えます。

- **ログ出力**: `%APPDATA%\MicSplitter\logs\` などのディレクトリに動作ログを出力します。
- **エラー通知**: バックグラウンド動作中にデバイス切断やストリーム初期化失敗などのエラーが発生した場合、Windowsのトースト通知等を利用してユーザーに知らせます。

## 5. 開発のフェーズ分け（マイルストーン）

- **フェーズ1: CUIプロトタイプの実装**
  - UIを持たず、ソースコード上でハードコードされたデバイス名を用いて、1入力→2出力＋モニターのルーティングが正しく行われ、音割れや遅延がないことを確認する。(cpal と ringbuf の結合テスト)
- **フェーズ2: 設定ファイルとオーディオエンジンの連携**
  - serde を用い、JSONファイルからデバイス名、トグル動作設定、モニター設定などを読み込んでストリームを構築・制御できるようにする。
- **フェーズ3: メインウィンドウGUIとタスクトレイUIの実装**
  - eframe (egui) を導入し、レベルメーターやフローチャートUI、デバイス選択などを行うメインウィンドウを実装します。
  - サウンド設定を開くボタンの実装。
  - tray-icon を導入し、出力状態に合わせて同梱のトレイアイコン画像（白、赤、緑、赤＆緑）を変化させるロジックを実装します。
  - ウィンドウを閉じる/最小化した際の非表示化および再表示ロジックを構築します。
- **フェーズ4: システム連携とスタートアップの統合（単一インスタンス・自動リカバリー）**
  - グローバルホットキーを導入し、アプリが非アクティブであってもトグル動作や個別ON/OFFができるようにします（設定変更UIも実装）。
  - Named Mutex による単一インスタンス制限の実装。二重起動検知時は既存プロセスへ再表示メッセージを送信し、新規プロセスは終了します。
  - 引数 `--autostart` の有無による起動時の動作分岐の実装。
  - 標準的な自動起動機能（レジストリRun等）の実装。
  - USBデバイスの接続切断、スリープ復帰時の自動リカバリー（再接続・ストリーム再構築）ロジックをオーディオエンジンに実装します。

## 6. 技術的な懸念事項と対策（重要）

- **サンプルレートとフォーマットの不一致**
  - 入力マイクが「48000Hz, 2ch」で、出力先の仮想ケーブルが「44100Hz, 2ch」などの設定になっている場合、そのままデータを流し込むとピッチが変わるか、アプリがクラッシュします。
  - **対策**:
    - アプリ側でデバイス読み込み時にサンプリングレートおよびフォーマットの不一致を検知した場合、音声転送を安全にブロックします。
    - メインウィンドウGUI上に、具体的なエラー内容と手順を示した警告メッセージを赤文字等で目立つように表示します。

- **チャンネル数の不一致**
  - 入力がモノラル（1ch）で出力先がステレオ（2ch）といったケース。
  - **対策**: モノラルからステレオになる際は、アプリ側で音声データを複製（同一サンプルを左右に配置）して出力ストリームに書き込み対応します。

- **クロックドリフトによる音飛び・ノイズ（経時的なズレ）**
  - 入力デバイスと出力デバイスのハードウェアクロックの微細な誤差により、長時間稼働した際にリングバッファが空になる（アンダーラン）か、溢れる（オーバーラン）が発生して「プチプチ」とした音飛び・ノイズやレイテンシの増加に繋がります。
  - **対策**:
    - バッファのアンダーラン発生時は、不足分のサンプル数に対して無音データ（ゼロ）を安全に補填します。
    - バッファのオーバーラン（遅延の蓄積）を検知した場合は、リングバッファ内の古い音声データを一定量破棄（最新フレームへシーク）し、遅延とノイズを抑える自動復帰バッファ制御ロジックを実装します。

- **WASAPIの排他（占有）モードによる競合**
  - 他のDAWソフトや通話アプリが対象デバイスを「排他モード」で占有している場合、本アプリはオーディオストリームの構築に失敗します。
  - **対策**:
    - デバイスの初期化処理時にエラーが発生した場合は、GUI上やトースト通知で、他アプリによる独占使用がされていないか確認を促す詳細なメッセージを表示します。

- **自動リカバリー機能（再接続・スリープ復帰）の実装**
  - デバイスの物理的な切断や、PCがスリープした後の復帰時に、オーディオストリームが破綻して音声が流れなくなります。
  - **対策**:
    - ストリームエラーの検知やポーリングにより、デバイスが見つかり次第自動でストリームを再構築します。

- **eframe (egui) と tray-icon のイベントループの共存**
  - eguiとタスクトレイ（`tray-icon`）のイベントループが干渉して、UIのフリーズやタスクトレイクリックに反応しない問題が発生しやすいです。
  - **対策**:
    - Tauriなどの別フレームワークは利用せず、eframeを前提として適切に非表示化とトレイ連携が行えるよう設計・検証を行います。
